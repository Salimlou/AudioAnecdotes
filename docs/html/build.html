Building Audio Anecdotes

 

This document describes how to build Audio Anecdotes example code including the PABLIO audio library. This document assumes that the source code has already been installed to a writeable file-system on your computer from the CD using either the provided installer or the manual tarball installation method and that a POSIX tool chain is available on your machine (Linux and MacOSX distributions include these tools, for Windows the Cygwin environment is distributed on the Audio Anecdotes CD and is installed by default)

 

Audio Anecdotes’ build process is a traditional recursive build making use of make and GNU Autotools to provide an automatic build on our supported platforms: (Windows/Cygwin, Linux and MacOSX).  Audio Anecdotes and PABLIO source code is largely portable and will likely build on other platforms. PortAudio, our audio portability layer, is platform specific but has been ported to a variety of popular platforms. For a complete build two sequential traversals (a configuration traversal and a build traversal) are made, both are initiated by the top level Makefile. 

 

The first traversal is controlled by the config.mak files found in each directory.  These makefiles (called config.mak so as not to conflict with the makefiles used in the second traversal) are responsible for calling the configure scripts which determine the nature of the platform you are running (the compiler used, filename extensions for objects, .etc) and use this information to translate the generic Makefile.in files to Makefiles specific to your platform.

 

The second traversal then recursively calls these makefiles created in the first stage to first build the Audio Anecdotes examples.  The build traversal first creates libraries in the src directory and then the audio examples in Content.

 

Building is simple: In your favorite command shell (via cygwin on Windows), cd to the directory you installed Audio Anecdotes to and then simply type make in the Volume1 directory everything should build automatically:

 

            cd /usr/local/AudioAnecdotes/Volume1

      make

 

 

Temporary files may be removed using the following command:

           

            make clean

 

 

The distribution may be clobbered (All temporary files, executables and copied headers will be removed) by using the following command:

 

            make distclean

 

NOTE: This will remove the executable examples causing the tour not to function until either a successful build is completed or the CD is re-installed!

 

 

To get rid of Makefiles (generated from Makefile.in files):

 

            make config-nomake

 

 

To remove the entire build environment configuration:

 

            make config-distclean

 

 

Examples are distributed in source form to encourage the reader to experiment with the code. Consequently the above mentioned commands will also work in each ‘leaf’ directory.  For instance the user would type make in a directory where a source file was modified to cause the executable to be automatically rebuilt. The user is responsible for understanding the inter directory dependencies if they wish to build in specific directories only.  Issuing make at the top level, volume1, directory is always safe if a little slower.

 

Commands that control configuration may also be individually issued in directories other than the top level; however they need to explicitly specify the config.mak file explicitly as in the following examples.

 

To configure the build environment:

 

            make -f config.mak

                        This will build configure scripts if necessary

                        run them build makefiles in all relevant subdirectories

 

To remove all Makefiles in the build environment:

 

            make -f config.mak nomake

 

 

To remove the build environment configuration:

 

            make -f config.mak distclean
